# 🔐 Security Audit Project – Botium Toys (Simulated)

## 📌 Project Overview

This project presents a security audit conducted for a simulated organization, Botium Toys, as part of a cybersecurity training program. The purpose of this audit is to identify security risks, evaluate existing controls, and assess compliance gaps to improve the organization’s overall security posture.

---

## 🎯 Scope and Goals

### Scope

The audit covers the entire security program of the organization, including employee devices, internal networks, business systems, and data management practices. It also evaluates existing security controls and compliance measures.

### Goals

* Assess current organizational assets
* Evaluate security controls
* Identify vulnerabilities and compliance gaps
* Recommend improvements to enhance security posture

---

## 🧩 Current Assets

* Employee devices (desktops, laptops, smartphones)
* Internal network and internet access
* Business systems (ecommerce, database, accounting)
* Data storage and retention systems
* Legacy systems requiring maintenance
* Physical assets (storefront, warehouse, CCTV systems)

---

## ⚠️ Risk Assessment

### Risk Description

The organization has inadequate asset management and lacks several critical security controls. Additionally, compliance with industry standards and regulations is incomplete.

### Risk Score

**8/10 (High Risk)**

---

## 🔍 Key Findings

* No encryption for sensitive customer data
* Weak password policies
* No intrusion detection system (IDS)
* No backup or disaster recovery plan
* Lack of access control (no least privilege enforcement)
* All employees have broad access to sensitive data

---

## 🛡️ Existing Controls

* Firewall configured with security rules
* Antivirus software installed and monitored
* Physical security measures (locks, CCTV, fire detection systems)

---

## ❗ Missing Controls

* Least Privilege access control
* Separation of Duties
* Intrusion Detection System (IDS)
* Data encryption
* Backup and disaster recovery
* Password management system

---

## 📜 Compliance Gaps

### PCI DSS

* No encryption for credit card data
* Excessive access to sensitive information
* Weak password management

### GDPR

* Data is not fully secured
* Asset classification is incomplete
* Breach notification plan is implemented

### SOC 1 / SOC 2

* Weak access control policies
* Data confidentiality not ensured
* Data integrity maintained

---

## 🛡️ Recommendations

* Implement encryption for sensitive data
* Enforce least privilege access control
* Deploy an intrusion detection system (IDS)
* Establish backup and disaster recovery plans
* Strengthen password policies and management
* Improve asset inventory and classification

---

## 🧾 Conclusion

The audit reveals significant security gaps that expose the organization to a high level of risk. Immediate action is required to implement proper security controls and ensure compliance with industry standards.
